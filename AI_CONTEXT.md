# AI Context — 임베디드 TIL

이 파일을 대화 시작 전에 붙여넣으면 맥락 파악 가능.

---

## 나는 누구

- iOS 개발 경험 있는 주니어 개발자, 임베디드 SW로 전환 중
- 목표: STM32 bare-metal 포트폴리오 완성 → 임베디드/전장 취업

---

## 이 저장소 목적

임베디드 공부하면서 배운 것 마크다운으로 기록. 코드 없음, 글만.

---

## 학습 순서

```
1. C 언어 (포인터, 비트 연산, volatile, 구조체)
2. STM32 GPIO (MODER, ODR, IDR 레지스터 직접 제어)
3. STM32 UART (printf 리다이렉트, 디버깅)
4. STM32 I2C (레지스터 직접, HAL 미사용)
5. STM32 CAN (루프백 테스트)
```

---

## 폴더 구조

```
TIL/
├── c-lang/       ← C 언어 기초 (학습 노트)
├── stm32/
│   ├── gpio/
│   ├── uart/
│   ├── i2c/
│   └── can/
├── cs/           ← CS 기초 (인터럽트, 메모리 등)
└── 면접대비/     ← 답변카드, 오답노트 (면접 직전 입에 붙이기용)
    ├── 답변카드/
    └── _오답노트.md
```

## 학습 → 면접 흐름

1. 학습노트 작성 (`c-lang/` 등)
2. `/make-card <노트>` → 답변카드 자동 추출
3. `/easy-quiz` → 답변카드 기반 부드러운 면접 연습


---

## 학습 노트 템플릿

- 새 학습 문서를 정리할 때는 `templates/학습노트.md` 형식을 기본 템플릿으로 사용한다.
- 주요 섹션은 `## 1. 제목` 형식으로 번호를 붙인다.
- 기본 콜아웃은 라벨 없이 사용한다.
- 실수하기 쉬운 내용과 예외만 `> **주의**`, `> **예외**` 라벨을 사용한다.
- 섹션 사이는 `---`로 구분한다.
- 마지막에는 `## 참고 자료` 섹션을 둔다.
